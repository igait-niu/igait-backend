5/// This module contains the pipeline submission endpoint for the API.
6/// 
7/// This endpoint receives results from the iGait pipeline running on the compute cluster.
8/// It is secured with a shared secret to ensure only authorized pipeline instances can submit.
9/// 
10/// # Arguments
11/// * `X-Pipeline-Secret` header: The shared secret for authentication
12/// * `output`: JSON-serialized Output struct (multipart field)
13/// * `archive`: Optional results.zip file (multipart field)
14/// 
15/// # Example cURL Request
16/// ```sh
17/// curl -v -X POST \
18///     -H "X-Pipeline-Secret: MEOWMEOWMEOW" \
19///     -F output=@output.json \
20///     -F archive=@results.zip \
21///     http://api.igaitapp.com/api/v1/pipeline/submit
22/// ```
23/// 
24/// # Testing
25/// For local testing instructions, see the [testing guide](../docs/testing.md).
26/// 
27/// # Potential Reasons for Failure
28/// * `401 Unauthorized` - Invalid or missing secret
29/// * `400 Bad Request` - Malformed data
30/// * `500 Internal Server Error` - Processing failed
31pub mod pipeline;

33/// This module contains the historical submissions endpoint for the API.
34/// 
35/// # Arguments
36/// * `uid`: The Google Firebase user ID. Use the Firebase JS SDK to obtain before submitting.
37/// * `entries`: The number of entries to return. This is an unsigned integer.
38/// * `start_timestamp`: The UNIX timestamp (seconds since the epoch date) to start the search from.
39/// * `end_timestamp`: The UNIX timestamp (seconds since the epoch date) to end the search from.
40/// * `result_type`: The type of result to return. This is a string - either `ASD` or `NO ASD`.`
41/// * `include_original`: Whether to include the original video files in the response.
42/// * `include_skeleton`: Whether to include the skeleton files in the response.
43/// 
44/// # Example cURL Request
45/// ```sh
46/// curl -v -F user_id=hiibolt -F entries=2 -F start_timestamp=173
47/// 6628724 -F end_timestamp=3736628726 -F result_type='NO ASD' -F inc
48/// lude_skeleton=true -F include_original=true http://api.igaitapp.com/
49/// api/v1/historical_submissions
50/// ```
51/// 
52/// # Potential Reasons for Failure
53/// See each individual function for potential reasons for failure.

56/// This module contains the job upload endpoint for the API.
57/// 
58/// # Arguments
59/// * `uid`: The Google Firebase user ID. Use the Firebase JS SDK to obtain before submitting.
60/// * `age`: The age of the user in `MM/DD/YYYY` format.
61/// * `ethnicity`: The ethnicity of the user.
62/// * `sex`: The sex assigned to the patient at birth.
63/// * `height`: The height of the user in `ft'in"` format.
64/// * `weight`: The weight of the user in pounds.
65/// * `email`: The email address of the user of which to send emails to.
66/// * `front_file`: The front video file. Needs to be a file compatible with OpenPose.
67/// * `side_file`: The side video file. Needs to be a file compatible with OpenPose.
68/// 
69/// # Example cURL Request
70/// ```sh
71/// curl -v -F fileuploadfront=@test.mp4 -F fileuploadside=@test.mp4 /
72///     -F uid=curlplaceholder -F age=18 -F ethnicity=Caucasian /
73///     -F email=me@hiibolt.com -F sex=M -F height="5'10" -F weight=120 / 
74///     http://api.igaitapp.com/api/v1/upload
75/// ```
76/// 
77/// # Potential Reasons for Failure
78/// See each individual function for potential reasons for failure.
79/// 
80/// # Notes
81/// <div class="warning">This route takes a long time to process. Make users aware of this on the frontend.</div>

84/// This module contains the assistant endpoint for the API.
85/// 
86/// Currently, this works by proxying requests to the iGait Assistant API.
87/// 
88/// This is done so due to the limitations of Firebase Auth expecting authentication via the `Bearer` header.
89/// Unfortunately, the WSS standard does not support this, so we instead extract it from an initial request and re-proxy to a now-authenticaed secure websocket.
90/// This approach is certainly not ideal, but it is indeed highly secure. 
91/// In fact, it is significantly more secure than the WSS standard itself, which does not support authentication at all.
92/// 
93/// For more information on usage, see the iGAIT frontend codebase.
94pub mod assistant;
